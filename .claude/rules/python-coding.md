# Python コーディングルール

## コード品質

- すべてのコードに型ヒントを必須とする
- パブリック API には必ず docstring を付ける
- 関数は集中して小さく保つ
- 既存のパターンを正確に踏襲する
- 行の最大長は 88 文字まで

## テスト

- テストフレームワーク: `uv run --frozen pytest`
- 非同期テストは `asyncio` ではなく `anyio` を使用
- カバレッジはエッジケース・エラーも含めてテストする
- 新機能には必ずテストを追加する
- バグ修正にはユニットテストを追加する
